# 技能结构清单

本文档定义了各类技能的标准结构，用于确保生成技能的一致性和完整性。

## k_ 技能（任务技能）结构

### 必需章节

```markdown
---
name: k01_init_project
description: 简短描述（50字以内）
---

# 技能名称

## 角色定位
（一句话说明技能的职责）

## 前置条件
- 环境要求
- 依赖工具/库
- 必要的配置

## 执行流程
### 步骤 1：[步骤名称]
[详细说明]

### 步骤 2：[步骤名称]
[详细说明]

## 预期输出
- 文件清单
- 配置说明
- 验证方法

## 常见问题
| 问题 | 解决方案 |
|-----|---------|
| ... | ... |

## 参考来源
- 基于: p_xxx 或 u_xxx（如有）
- 任务: k01
```

### 可选章节

- **最佳实践**：推荐做法
- **注意事项**：需要避免的坑点
- **相关资源**：文档链接、参考实现

## u_ 技能（用户经验）结构

### 必需章节

```markdown
---
name: u_react_hooks
description: 用户经验技能描述
---

# 技能名称

## 经验概述
[经验背景和适用场景]

## 执行流程
### 1. [步骤]
[详细说明]

## 最佳实践
| 实践 | 说明 |
|-----|------|
| ... | ... |

## 已知坑点
| 问题 | 解决方案 |
|-----|---------|
| ... | ... |

## 关联项目
- k01: [项目名称] ([日期])
```

## p_ 技能（验证技能）结构

与 k_ 技能结构相同，但需要额外添加：

### p_ 技能特有章节

```markdown
## 验证信息
- 来源任务: k01
- 验证时间: [ISO 8601 格式]
- 成功率: [0-1]
- 复用次数: [数字]
```

## 内置技能结构

内置技能（user-profile, commander, skill-generator）需要：

### 必需章节

1. **YAML 前置元数据**
2. **角色定位**：明确职责边界
3. **职责边界表**：应该做 / 不应该做
4. **与其他技能的关系**：ASCII 关系图
5. **调用方式**：如何调用该技能
6. **处理流程**：详细的执行步骤
7. **错误处理**：异常情况处理
8. **相关参考**：指向其他文档的链接

## 技能质量检查清单

生成技能时，确保：

### 基础要素
- [ ] 包含 YAML 前置元数据
- [ ] 角色定位清晰
- [ ] 执行流程完整可执行
- [ ] 包含错误处理

### 内容质量
- [ ] 步骤描述具体明确
- [ ] 包含预期输出说明
- [ ] 提供验证方法
- [ ] 包含参考来源

### 一致性
- [ ] 命名符合规范
- [ ] 结构与同类技能一致
- [ ] 使用标准化的章节标题
- [ ] 格式符合 Markdown 规范
